## What's in here

Compares three ways to write a CSV out as a Parquet file: pandas, pyarrow, and polars. All three produce a valid Parquet on disk — but with different sizes and different ergonomics.

- `prepare_parquet.py` — the demo script the video walks through
- `prices.csv` — sample input data (14 tickers × OHLCV daily)

## Run it

```bash
pip install -r requirements.txt
python prepare_parquet.py
```

## What it shows

1. `pd.read_csv(...).to_parquet(...)` — pandas one-liner. Uses pyarrow under the hood.
2. `pq.write_table(pa_csv.read_csv(...), ...)` — pyarrow native, no pandas roundtrip.
3. `pl.read_csv(...).write_parquet(...)` — polars one-liner. Rust-backed end-to-end.

All three write a column-store binary format that's ~5-10× smaller than the CSV and ~50× faster to read.
